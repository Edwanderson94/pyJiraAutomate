# QA

Espaco dedicado para validacoes manuais e evidencias de testes do projeto `pyJiraAutomate`.

## Objetivo

Organizar os testes antes e depois das implementacoes para comparar comportamento, registrar evidencias e acompanhar melhorias.

## Estrutura

- `before/`: testes, notas e evidencias coletadas antes de uma mudanca.
- `after/`: testes, notas e evidencias coletadas depois de uma mudanca.
- `checklists/`: roteiros reutilizaveis de validacao.

## Como usar

1. Antes de alterar uma funcionalidade, registre o estado atual em `before/`.
2. Depois da implementacao, registre o resultado em `after/`.
3. Use os arquivos em `checklists/` para manter o mesmo criterio de validacao.
